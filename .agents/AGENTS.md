# Workspace Guidelines

## Documentation Rule
Bất cứ khi nào code một tính năng mới hoặc một module mới, BẮT BUỘC phải tạo/cập nhật các tài liệu liên quan đến tính năng đó.
- Các tài liệu này phải được đặt trong thư mục `docs/`.
- Phân loại tài liệu vào các thư mục con tương ứng, ví dụ như `docs/develop/` (cho đặc tả kỹ thuật, API) hoặc `docs/guideline/` (cho hướng dẫn sử dụng, convention).
